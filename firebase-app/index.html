Firebase Auth + Realtime Database

Controle de acesso por cargo

Aplicação web com autenticação, modelagem em árvore JSON e controle de acesso por Firebase Security Rules para os cargos admin e user.

Status do Firebase
Firebase ainda não inicializado.

1. Configuração

Cole ou carregue automaticamente a configuração do seu projeto Firebase.

Dica: para esta aplicação, a config pode ficar no front-end. O que realmente protege o banco são as Security Rules.

2. Cadastro e login

Crie contas com os dois cargos e teste as permissões.

Observação acadêmica: nesta demo, o cargo pode ser escolhido na interface. Em um sistema real, o cargo admin não deve ser definido pelo cliente.

6. Regras aplicadas no projeto

Resumo visual do comportamento esperado das Firebase Security Rules.

/users/{uid}

Leitura e escrita permitidas apenas para o próprio usuário autenticado.

/admin-data

Leitura e escrita permitidas apenas para usuários com cargo admin.

Validação

Os campos esperados são controlados por validações nas regras do Realtime Database.
